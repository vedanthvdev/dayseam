## 3. First-run / upgrade path observations

Free-form notes on anything the boot sequence, the Reconnect flow, or the per-kind renderer did that surprised the dogfood owner on Day 1. Focus areas:

- **Migration `0005_secret_ref_index.sql`** — does `idx_sources_secret_ref` appear in `sqlite_master` after boot? Any visible latency delta on source-list hydration? (Expected answer: no, since the index only helps at ~1000+ row counts and Dayseam's live installs carry 5–20.)
- **Per-kind section rendering edge cases** — GitLab-only day (only `Commits` visible, MRs rolled into it), Atlassian-only day (only `Jira issues` + `Confluence pages` visible), empty day (all sections suppressed except the legacy `## Commits` "No tracked activity" fallback), mixed day (all three visible and ordered).
- **Reconnect chip affordances** — is the chip visually distinguishable on a source that actually needs reconnection vs a healthy one? Does the dialog's pre-fill round-trip through `AddAtlassianSourceDialog` without losing the original label or the Journey A (shared-PAT) affinity?
- **SerdeDefaultAudit waivers** — did anything compile-warn / compile-error in release mode that passed in dev? (Expected answer: no, since the derive is pure proc-macro scaffolding with the same behaviour across profiles.)
- **Cross-source enrichment on mixed day** — Jira transitions triggered by GitLab MRs still carry the `(triggered by !<iid>)` annotation? The DAY-88 temporal-ordering fix still behaves as expected when an MR and a transition are hours apart vs minutes apart?
